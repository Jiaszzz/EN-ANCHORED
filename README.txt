【這個資料夾裡有什麼】

album/
  index.html      ← 網頁主結構
  style.css       ← 所有樣式（紫色玻璃風格）
  config.js       ← 歌曲清單，換歌只改這個檔案
  coverflow.js    ← 封面滑動效果
  audio.js        ← 播放邏輯（含預載下一首、鎖定畫面控制）
  player.js       ← 串接以上所有模組
  sw.js           ← 離線快取，讓現場網路不穩也能順利播放
  covers/01.jpg ~ 07.jpg  ← 目前是「佔位圖」，請替換成你的封面
  songs/01.mp3  ~ 07.mp3  ← 目前是「5 秒靜音佔位音檔」，請替換成你的歌曲

【現在就可以先測試】

這個資料夾現在整組上傳到 GitHub Pages 就可以直接運作
（用佔位圖跟靜音檔），可以先確認整體效果、滑動、UI 都正常，
再換成正式的歌曲跟封面。

【換成正式歌曲的方法】

1. 把你的 7 首 MP3，改檔名為 01.mp3、02.mp3 ... 07.mp3
   （檔名必須完全對應，不能有中文或空白）
2. 把你的 7 張封面圖，改檔名為 01.jpg、02.jpg ... 07.jpg
   （如果你的圖是 png，記得把 config.js 裡對應的副檔名也改成 .png）
3. 到 GitHub 網頁介面，把新檔案上傳覆蓋 songs/ 跟 covers/
   資料夾內同名的舊檔案即可
4. 打開 config.js，把每首歌的 title（歌名）、artist（演唱者）
   改成正確資訊，存檔

【換完之後記得做這件事】

因為加了離線快取（sw.js），瀏覽器可能還記得舊版的佔位音檔。
第一次上傳正式歌曲後，建議：
- 在手機瀏覽器「清除網站資料」再重新整理一次，或
- 用「無痕視窗」重新打開網址測試一次
確保拿到的是最新版本，而不是被快取住的舊佔位檔。

【重新掃 NFC 的行為】

已依你的要求設定為：每次打開都從第一首重新開始播放，
不記憶上次播放位置，確保每次都是完整的體驗。

【新增：字型與開場婚紗照】

fonts/ 資料夾裡是三款字型的檔案（Yellowtail 英文手寫花體、
Ma Shan Zheng 中文毛筆字、Playfair Display 優雅襯線），
fonts.css 是對應的設定檔，不需要修改。

wedding-bg.jpg 目前是「佔位圖」，開場畫面現在會整張顯示這張圖當背景。
請把你們的婚紗照，改檔名為 wedding-bg.jpg，覆蓋這個佔位檔即可
（建議直式構圖、至少 1200x1600 以上解析度，效果會最好）。
